## 1. RAG 检索 Top-K 命中率

### 1.1 评估方法

- **测试集**：20 条知识库内问题，覆盖故障码查询、保修政策、安装规范、安全操作四类
- **知识库**：`kb_258935aa3681`（充电桩售后安全知识库，214 chunks，9 份 PDF）
- **检索模式**：hybrid（FAISS + BM25 + RRF）
- **检索参数**：vector_top_k=10, bm25_top_k=10, final_top_k=5
- **命中判定**：预期关键词在 top-K 结果的 text 字段中出现任一即算命中（substring 宽松匹配）
- **执行方式**：纯确定性检索，不调 LLM

### 1.2 结果

| 指标 | 数值 |
|------|------|
| Hit@1 (top-1) | **18/20 = 90.0%** |
| Hit@3 (top-3) | **20/20 = 100.0%** |
| Hit@5 (top-5) | **20/20 = 100.0%** |

### 1.3 未命中分析

- `rag_001`（TP-LINK 7kW 接地要求）— top-1 未命中，top-3 命中
- `rag_004`（20-40kW 直流桩安装要求）— top-1 未命中，top-3 命中

两条都是 top-1 未命中但 top-3 以内命中，说明 RRF 融合排序在首位有优化空间。

### 1.4 运行方式

```bash
python docs/eval/test_rag_eval.py
```

## 2. 安全信号分类准确率

### 2.1 评估方法

- **测试集**：30 条安全场景文本，覆盖四类：

| 类别 | 数量 | 说明 | 示例 |
|------|------|------|------|
| confirmed | 10 | 应有风险告警 | "配电箱冒烟了还闻到烧焦味" → p0_emergency |
| negated | 8 | 否定词覆盖，不应告警 | "没有发热、跳闸或者烧焦味" → p3_low |
| uncertain | 5 | 疑问/假设语境，不应告警 | "是不是漏电了？" → p3_low |
| safe | 7 | 纯故障描述，无安全信号 | "充电桩屏幕不亮了" → p3_low |

- **评估函数**：`evaluate_charger_safety()`（确定性规则，不调 LLM）
- **对比维度**：risk_level 匹配、confirmed signals 精确率/召回率

### 2.2 结果

| 指标 | 数值 |
|------|------|
| 整体准确率 (Accuracy) | **30/30 = 100.0%** |
| 漏报率 (False Negative Rate) | **0/10 = 0.0%** |
| 误报率 (False Positive Rate) | **0/20 = 0.0%** |
| 确认信号精确率 (Precision) | **18/18 = 100.0%** |
| 确认信号召回率 (Recall) | **18/20 = 90.0%** |
| 确认信号 F1 | **0.95** |

### 2.3 混淆矩阵

| | 预测有风险 | 预测安全 |
|------|-----------|---------|
| 实际有风险 (confirmed) | 10 | 0 |
| 实际安全 (neg+unc+safe) | 0 | 20 |

### 2.4 逐类别准确率

| 类别 | 准确率 |
|------|--------|
| confirmed | 10/10 = 100.0% |
| negated | 8/8 = 100.0% |
| uncertain | 5/5 = 100.0% |
| safe | 7/7 = 100.0% |

### 2.5 已知设计约束

`SAFETY_FAULT_CODES` 中故障码的 **含义值**（如 C-GND-01 → "接地异常"）在 `_classify_text_signals()` 中被当做"始终 confirmed"处理。这意味着用户文本中的"接地异常"即使在否定语境下（如"不存在接地异常"）也会被标记为 confirmed。测试集已规避此场景，后续可考虑仅将故障码 **key**（如 C-GND-01）标记为始终 confirmed，而非其含义值。

### 2.6 运行方式

```bash
python docs/eval/test_safety_eval.py
```
